# Documentation

Docs split by task instead of one long handbook.

## Read By Goal

- First local run: [`setup.md`](setup.md)
- Need command syntax: [`commands.md`](commands.md)
- Need code map: [`architecture.md`](architecture.md)
- Need dev workflow or test rules: [`development.md`](development.md)
- Need runtime, Docker, env, or deployment notes: [`operations.md`](operations.md)
- Need to fork repo into another Mezon bot: [`forking-guide.md`](forking-guide.md)

## Audience Guide

- Contributors: start with [`development.md`](development.md)
- Operators: start with [`operations.md`](operations.md)
- Coding agents: start with [`architecture.md`](architecture.md), then read target page only
- Fork authors: start with [`forking-guide.md`](forking-guide.md)

## Source Of Truth

Docs reflect current repo state. If behavior changes, update matching page in `docs/` and keep root `README.md` short.
